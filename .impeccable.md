# Alkilo Design System

## Design Context

### Users
**Primary audience**: Cubanos locales buscando alquilar propiedades dentro de Cuba.

**Context of use**:
- Conexión de datos móviles limitada — la app debe cargar rápido y consumir pocos datos
- Usuarios buscan alojamiento para viajes internos, reuniones familiares, escapadas de fin de semana
- Mezcla de edades y niveles técnicos — la interfaz debe ser intuitiva y clara

**User goals**:
- Encontrar propiedades disponibles rápidamente
- Ver fotos y detalles sin esperar cargas largas
- Reservar con confianza (transacciones claras y seguras)
- Hosts: publicar y gestionar sus propiedades fácilmente

### Brand Personality
**Three words**: Confiable, profesional, accesible

**Voice & tone**:
- Serio pero no frío — transmite confianza sin ser distante
- Directo y claro — respeta el tiempo del usuario
- Sin florituras — cada elemento tiene propósito

**Emotional goals**:
- Generar confianza para transacciones (esto es dinero real)
- Sensación de calidad y seriedad
- Facilidad y claridad — nunca confusión

### Aesthetic Direction

**Visual tone**: Minimalismo cálido

**Style pillars**:
1. **Limpio y despejado** — Espacios generosos, sin ruido visual
2. **Neutros cálidos** — Arena, beige, terracota suave, blancos cremosos
3. **Tipografía clara** — Jerarquía fuerte, legibilidad perfecta
4. **Fotografía como hero** — Las propiedades son el protagonista, no la UI

**Color palette direction**:
- **Primary**: Terracota cálido o marrón arena — serio pero no frío
- **Surface**: Blancos cremosos, beiges muy suaves (no grises puros)
- **Text**: Marrones oscuros sobre fondos claros (mejor contraste que negro puro)
- **Accent**: Un toque de color funcional para CTAs — coral suave o terracota más intenso

**Theme**: Light mode como default (más apropiado para app de propiedades), dark mode como alternativa (respeta preferencia del sistema)

**Typography direction**:
- Sans-serif geométrica o humanista para cuerpo (clara, moderna)
- Pesos contrastantes para jerarquía (light para cuerpo, semibold/bold para títulos)
- Tamaños generosos — legibilidad en conexiones lentas donde las imágenes cargan después

**Anti-references (what NOT to look like)**:
- Airbnb genérico con cards idénticas
- Gradientes azul-púrpura de AI
- Dark mode con glows neón
- Interfaces sobrecargadas de información

### Design Principles

1. **Performance es UX** — Cada KB cuenta. Imágenes optimizadas, animaciones mínimas, estados de carga elegantes.

2. **Confianza visual** — Elementos consistentes, espaciado predecible, sin sorpresas. La seriedad genera seguridad.

3. **Fotografía primero** — Las propiedades se venden con fotos. La UI se aparta para que las imágenes brillen.

4. **Jerarquía implacable** — Una acción principal por pantalla. El usuario siempre sabe qué hacer.

5. **Accesibilidad práctica** — Contraste alto, touch targets generosos, texto legible. Cuba tiene usuarios de todas las edades.

### Technical Constraints

- **Platform**: Kotlin Multiplatform (Android, iOS, Web, Desktop)
- **UI**: Compose Multiplatform con Material3
- **Architecture**: Clean Architecture + MVI
- **Performance priority**: Optimizar para datos móviles limitados
  - Lazy loading agresivo
  - Imágenes comprimidas
  - Animaciones sutiles (no decorativas)
  - Estados offline/error bien diseñados
